# Terzaghi — rapport d'écarts de fidélité d'interface (client ↔ plateforme)

> **Objet** : baseline « avant » et checklist de recette des écarts d'**interface**, de
> **fonctionnement** et d'**affichage** entre l'outil Terzaghi fourni par le client
> (`terzaghi_V13.html`) et notre portage web. Exigence client ferme : **ZÉRO écart**
> (règle titulaire « fidélité interfaces client », ADR 0014 « zéro écart »).
> **Ce rapport ne corrige rien** — la correction sera un lot séparé sur cette base.

- **Doctrine appliquée (ADR 0014, rescope « zéro écart »)** : tout ce que l'outil client
  **AFFICHE** est exposable — le confidentiel est le **CODE** moteur, pas les valeurs.
  Un masquage §8 d'une valeur affichée par le client est donc **lui-même un écart**
  (statut `MASQUÉ-§8`, à corriger). Seule exception actée : localisations de nœuds EF
  du radier (sans objet ici). NB : l'élargissement effectif de la whitelist
  `ple*/qce/kp…` reste une **décision explicite expert + titulaire** (mémoire
  `roadsen-engine-output-whitelist-ple-qce`) — le présent rapport la **déclenche**, il
  ne la prend pas.

> **Portée honnête** : l'équivalence NUMÉRIQUE du portage moteur est couverte par les
> tests d'équivalence (`engine.equivalence.test.ts`, harnais golden-master). Le présent
> rapport porte sur la **fidélité d'INTERFACE et d'AFFICHAGE bout-en-bout**.

## Synthèse : verdict global

**La structure de saisie est globalement fidèle, la restitution des résultats ne l'est
pas.** Contrairement à GEOPLAQUE (écart de paradigme total), Terzaghi côté client est un
formulaire 2 colonnes → notre page reproduit correctement les 3 cartes de saisie
(sélecteurs segmentés, tableaux sondage/charges, guides α en modal, champs conditionnels
talus/c–φ/forme). **Les écarts majeurs sont concentrés côté résultats** :

1. **Onglet Coupe** : le client dessine une **vraie coupe technique live** (courbes
   pl\*/E_M ou qc en échelle log, couches, zone d'influence 1,5·B, De, nappe cotée,
   variante labo) ; nous affichons un **schéma statique** B/D/nappe sans aucune courbe.
2. **Onglet Vérifications** : le client déroule un **calcul pas-à-pas complet** avec
   formules substituées (11 étapes portance + glissement + tassement + raideurs) ; nous
   affichons une **table plate** d'une dizaine de lignes. Une grande partie des
   intermédiaires affichés par le client (ple\*, De, kp, qnet, u/q0/σ′v0, Ec/Ed, Kv/Kh/Kθ…)
   est **non whitelistée** dans `TerzaghiOutputSchema` → `MASQUÉ-§8` = écarts ADR 0014.
3. **Note de calcul** : document structuré complet chez le client (hypothèses, sondage,
   contraintes, chapitres par cas, synthèse, figure de coupe, impression navigateur) ;
   chez nous un tampon CONFORME + les mêmes tables plates.
4. **Workflows absents** : Exemple fictif, Import/Export .json, recalcul live,
   validation visuelle des champs, affichage des avertissements moteur.

## Classement des écarts

### MAJEURS

1. **Coupe sans profil** (E2, E4, E5, E7, E8) : aucune courbe pl\*/E_M/qc, pas de zone
   1,5·B, pas de De, pas de variante labo, pas de légende. L'onglet par défaut du client
   est une figure d'ingénierie ; le nôtre un pictogramme.
2. **Déroulé de calcul absent / masqué §8** (F7–F15, G4, G5) : le client montre chaque
   étape avec valeurs substituées (ple\*, De, kp avec formule de courbe, iδβ, qnet, qref,
   u/q0/σ′v0, Ec/Ed, sc/sd, C1/C2/C3, Kv/Kh/Kθ…). Notre contrat serveur
   (`TerzaghiOutputSchema`) ne whiteliste presque aucun de ces champs → double chantier :
   **décision d'élargissement de whitelist (expert + titulaire)** puis rendu front.
3. **Note de calcul vidée** (G1–G10) : pas d'hypothèses, pas de table sondage, pas de
   chapitres par cas, pas de synthèse dédiée, pas de figure de coupe, pas d'impression.
   G2/G3 (hypothèses + sondage = **entrées**) sont corrigeables **sans toucher au
   contrat**.
4. **Avertissements moteur non affichés** (F1, G7) : les warnings normatifs redactés
   existent dans la sortie serveur et sont simplement ignorés par la page — perte
   d'information de sécurité d'ingénierie.
5. **Workflows de fichier absents** (A3, A4, A5, A7, I1, I3) : Exemple fictif,
   Import/Export .json, impression, recalcul live, validation visuelle.

### MOYENS

- Mode labo non conforme (B6, B23) : sondage non masqué, UI c–φ non forcée.
- α constant sur la hauteur absent (B8, H2) + bouton guide bas de bloc (B9).
- Tassements en m au lieu de cm (F18) ; libellés d'états abrégés (D2) ; hint /ml non
  dynamique (D3) ; défaut 1 cas ELS_QP vs 2 cas (D6) ; ν pré-rempli 0.33 vs vide (B20).
- Pas de regroupement par cas ni rappel des charges dans Vérifications (F2, F3) ;
  capacité de référence sans déroulé ni tassement de référence (F17) ; messages
  d'erreur génériques (F20) ; notes par cas absentes (F19) ; **cas invalides
  silencieusement ignorés** (F21 — aucun message par cas rejeté).

### FAIBLES

- Libellés raccourcis (A1, B2, B17–B19, C5, C6, D5, B24) ; subnotes normatives absentes
  (B11, B12, B21, B25) ; nappe sans profondeur affichée (E6) ; intro/pied des modals
  condensés (H1, H3) ; Échap (H4) ; en-tête/pied de note réécrits (G1, G8).

## Les 5 écarts les plus visibles pour un utilisateur de l'outil client

1. **L'onglet Coupe ne montre plus son sondage** : il saisit pl\*/E_M (ou qc) et ne voit
   ni courbes en échelle log, ni couches, ni zone d'influence 1,5·B, ni De — juste un
   schéma générique B/D/nappe. C'est le premier écran qu'il voit.
2. **Plus aucun calcul déroulé** : là où l'outil client justifie chaque nombre (11 étapes
   de portance avec formules substituées, tassements décomposés sc + sd, raideurs
   Kv/Kh/Kθ), il ne reçoit qu'une table d'une dizaine de lignes de résultats finaux.
3. **La note de calcul n'est plus un document d'étude** : pas d'hypothèses, pas de table
   de sondage, pas de chapitres par cas, pas de synthèse, pas de figure, pas
   d'impression — remplacée par un tampon CONFORME et les mêmes tables.
4. **« Exemple fictif », Importer, Exporter ont disparu** : impossible de charger le cas
   de démonstration GEOFOND ou d'échanger un dossier .json comme il en a l'habitude.
5. **Rien ne bouge quand il tape** : le recalcul live permanent (coupe, vérifications et
   note toujours à jour, champs invalides marqués en rouge) est remplacé par un
   clic-serveur qui, en plus, affiche les tassements en mètres (0,05 m) là où son outil
   dit 5,00 cm — et ne lui montre plus les avertissements normatifs.
